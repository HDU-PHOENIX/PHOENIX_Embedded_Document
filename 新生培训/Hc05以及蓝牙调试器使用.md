# Hc05以及蓝牙调试器使用
> Hc05是一款常用的蓝牙模块。本文将介绍Hc05的基本使用方法以及如何通过蓝牙调试器进行调试，并与stm32单片机进行通信

在看本文档前，请先确保你已学会基本uart通信相关知识
## 1. Hc05简介
### 1.1 模式
Hc05有两种工作模式：数据模式和命令模式
- 数据模式：用于与其他蓝牙设备进行数据传输
- 命令模式：用于配置Hc05模块的参数，如名称、密码等
### 1.2 引脚
Hc05模块通常有以下引脚：
- VCC：电源正极，通常为3.3V或5V
- GND：电源负极
- TXD：串口发送引脚
- RXD：串口接收引脚
- KEY/EN：用于切换命令模式和数据模式
## 2. Hc05连接
### 2.1 硬件连接
将Hc05模块与stm32单片机连接，连接方式如下：
- Hc05 VCC连接到stm32的3.3V或5V电源
- Hc05 GND连接到stm32的GND
- Hc05 TXD连接到stm32的UART RX引脚
- Hc05 RXD连接到stm32的UART TX引脚
### 2.2 切换模式
- 进入命令模式：按住按键后再上电
- 进入数据模式：直接上电

## 2. AT模式配置Hc05
### 2.1 进入AT模式
1. 按住KEY/EN引脚
2. 给Hc05上电
3. 使用串口调试工具连接Hc05，波特率为38400bps
4. 发送"AT"命令，若收到"OK"响应，则进入AT模式
### 2.2 常用AT命令
| 命令         | 说明                     |
| :--: | :--: |
| AT | 回复OK则表示连接成功并进入AT模式 |
| AT+VERSION? | 查询模块版本信息 |
| AT+NAME? | 查询模块名称 |
| AT+NAME=NewName | 设置模块名称为NewName |
| AT+PSWD? | 查询模块密码 | 回复当前密码 |
| AT+PSWD=1234 | 设置模块密码为1234 |
| AT+UART? | 查询波特率 |
| AT+UART=115200,0,0 | 设置波特率为115200，无校验，1停止位 |
| AT+ROLE? | 查询模块角色 |
| AT+ROLE=0 | 设置模块为从机模式 |
| AT+ROLE=1 | 设置模块为主机模式 |
| AT+RESET | 重启模块 |
| AT+ORGL | 恢复出厂设置 |

这里建议修改模块名称和密码，以防止与其他人的蓝牙设备冲突。然后把波特率改为常用的115200
如果尝试用 AT+PSWD=xxxx 设置密码，却返回 FAIL，这可能是参数格式不对（密码通常 4 位，如 1234；有些需 6 位如 123456，无引号）。
名字不能用中文，否则是乱码

2. 点击新增的调试器，点击`通信设置`，设置通信协议
![](https://youke1.picui.cn/s1/2025/11/19/691d652d016bc.jpg)
3. 我做的示例有两个按钮和两个拨杆，故设置两个bool和四个char类型的数据
- bool类型：用于按钮的按下和松开
- char类型：用于遥控器的X轴和Y轴数据
![](https://youke1.picui.cn/s1/2025/11/19/691d6564b365d.jpg)
4. 返回，点击`编辑固件`，点击右上角'+'号创建两个按钮和两个拨杆。并绑定对应的数据，按钮绑定一个bool数据，拨杆绑定两个char数据拨杆

5. 完成后如图所示：

## 4. stm32接收蓝牙数据
### 4.1 配置cubemx
常规配置不提
1. 配置uart，注意波特率，停止位，校验位等
2. 配置dma接收数据，rx配置为循环模式
![DMA式cubemx](https://youke1.picui.cn/s1/2025/11/19/691d5a467d163.png)
3. **把uart的rx接口配置为上拉模式**，因为HC05模块的高电平在2.8v左右甚至更低，无法满足stm32的高电平输入要求，上拉后可以把电平拉高到3.3v，满足stm32的输入要求
![](https://youke1.picui.cn/s1/2025/11/19/691d710f85818.png)
### 4.2 接收数据代码
根据自己编写的通信协议，配置好对应的接收缓冲区，本实例中我的通信是8字节，故定义的是一个8字节的缓冲区
```c
int main(void)
{

  /* USER CODE BEGIN 1 */

  /* USER CODE END 1 */

  /* MCU Configuration--------------------------------------------------------*/

  /* Reset of all peripherals, Initializes the Flash interface and the Systick. */
  HAL_Init();

  /* USER CODE BEGIN Init */

  /* USER CODE END Init */

  /* Configure the system clock */
  SystemClock_Config();

  /* USER CODE BEGIN SysInit */

  /* USER CODE END SysInit */

  /* Initialize all configured peripherals */
  MX_GPIO_Init();
  MX_DMA_Init();
  MX_USART1_UART_Init();
  /* USER CODE BEGIN 2 */
  uint8_t rx_buff[8] = {0};
  HAL_UART_Receive_DMA(&huart1, rx_buff, sizeof(rx_buff));
  /* USER CODE END 2 */

  /* Infinite loop */
  /* USER CODE BEGIN WHILE */
  while (1)
  {
    HAL_Delay(100);
    /* USER CODE END WHILE */

    /* USER CODE BEGIN 3 */
  }
  /* USER CODE END 3 */
}
```
